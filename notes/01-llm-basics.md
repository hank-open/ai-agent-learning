# LLM 基础参数

## Temperature (0.0 - 1.0)
- 0.0: 完全确定性，总是选择概率最高的 token
- 0.5: 平衡，用于大多数任务
- 1.0: 最随机，创意任务

对话示例:
Q: 2 + 2 = ?
- temp=0.0: "4"（确定）
- temp=0.7: "4"（大概率）
- temp=1.0: 可能偶尔出错，但更有创意

使用建议:
- 数学/逻辑/代码 → temp=0.0
- 问答/总结 → temp=0.3~0.5
- 创意写作/头脑风暴 → temp=0.7~1.0

## Max Tokens
- 限制输出长度上限
- 更短 = 更便宜，但可能截断回答
- 需要根据任务类型调整：
  - 简单问答: 100~200
  - 代码生成: 500~1000
  - 长文写作: 2000+

## Top-P (Nucleus Sampling)
- 只从累计概率达到 P 的 token 集合中采样
- 0.9: 保留概率最高的 90% 的 token
- 通常不需要同时调整 temperature 和 top_p，选一个就够

## 成本计算 (claude-3-5-sonnet)
- 输入 token: $3.00 / 1M tokens
- 输出 token: $15.00 / 1M tokens
- 多轮对话时，每轮都需要把完整历史传入 → token 快速累积

## 关键结论
1. 生产环境优先用低 temperature（稳定性 > 创意性）
2. 设置合理的 max_tokens 上限，避免浪费
3. 多轮对话成本随轮次线性增长，注意控制上下文长度
